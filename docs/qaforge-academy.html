    Why this is correct

    The login form does not validate empty inputs. When a user presses Login without entering an email or password, no client-side error is raised and a request is still sent to the backend. Proper validation should block submission and prompt the user before any request is made.

    Always test the empty-state path first — it's the most common validation gap.
    Confirm the request is actually blocked, not just visually flagged.
    Error copy should tell the user what to fix, not just that something failed.
    Check server-side validation too — client-side checks can be bypassed.
